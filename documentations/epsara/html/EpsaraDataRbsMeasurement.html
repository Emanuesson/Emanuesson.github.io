Property Details

The “calculation-type” property

  “calculation-type”         EpsaraBsCalculationType

Determines the calculation type. Usually Epsara uses the full analytic-calculation. However this might provide a higher precision but has the disadvantage of extraordinary long calculation times.

Flags: Read / Write

Default value: EPSARA_BS_CHUS_TYPE

The “depth-granularity” property

  “depth-granularity”        gdouble

the minimum resolved depth-difference in the calculation of energy-losses determines the accuracy of the simulation, but is always a trade-off to the performance.

Flags: Read / Write

Allowed values: >= 0

Default value: 1

The “ini-precision-in” property

  “ini-precision-in”         gdouble

the initial precision, that will be applied to the integration of the ion-trajectory on the invard-path.

Flags: Read / Write

Allowed values: >= 0

Default value: 0.005

The “ini-precision-out” property

  “ini-precision-out”        gdouble

the initial precision, that will be applied to the integration of the ion-trajectory on the out-path.

Flags: Read / Write

Allowed values: >= 0

Default value: 0.005

The “max-precision-in” property

  “max-precision-in”         gdouble

the maximum precision, that will be applied to the integration of the ion-trajectory on the invard-path.

Flags: Read / Write

Allowed values: >= 0

Default value: 1e-08

The “max-precision-out” property

  “max-precision-out”        gdouble

the maximum precision, that will be applied to the integration of the ion-trajectory on the out-path.

Flags: Read / Write

Allowed values: >= 0

Default value: 1e-08

The “suppress-dispersion” property

  “suppress-dispersion”      gboolean

suppresses all folding calculation with energy dispersion effects (e.g. E-straggling, beam-dispersion, detector-resolution). Is effective to accelerate the calculation time.

Flags: Read / Write

Default value: FALSE
