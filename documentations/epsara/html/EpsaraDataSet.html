Description

An analytical experiment always contains a whole bunch of intormation. In many cases these informations are linked in a highly complex way. The idea of the EpsaraDataSet-class is to bundle this information together in a way, that all relevant information is safed and the experimental questions are described completely. Only then a computer-algorithm is able to solve a physical problem.

Beside the experimental information, described with the EpsaraDataMeasurement-instances, a EpsaraDataSet-object contains information about the fitting process, such as a fitting-schedule, the fitting-algorithm-type and the optimum-point as well as the estimation of error-boundaries. It also contains a global random-number-generator, that should be used in any calculation, linked to a data-set. Only then it is possible to prevent correlations between several instances of the same pseudo-random-number-generator.

Adds one property to the list of parameters that will be used for global optimization (fitting). The property_name and objects_list build one entity, meaning one degree of freedom for the optimisation-kernel. The order of adding parameters with this function determines the logical order of parameters used by the data_set in optimization.
