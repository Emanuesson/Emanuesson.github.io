LibGDV

Description

Working in a scientific environment requires a fast, open and simple approach in software-design that is flexible enough to get extended to a full scale project in the end. The gtk+ library offers many of these features in terms of UI-programming and the underlying GLib and Gobject-framework is a powerful approach to use object-oriented programming (OOP) mostly independent of the available platform or programming language. Despite the advantages, most free scientific codes do not apply Gtk+ even in applications were the integration would be highly preferable. Gdv tries to provide a part of the functionality to make Gtk+ a little more interesting for people that are looking for a way to observe and control technical environments.

I startet to work on gdv because I was pretty annoyed by the options that are available for the visualisation of scientific and technical data, such as forcing Gnuplot within a GtkWidget-instance (this approach limited to systems with a working X11-platform), or using really old libraries such as GtkExtra that are now so outdated that they require functions deprecated 17 years ago, or a rather specialized library such as GOffice that reimplements a lot of the overhead that has been removed or improved in Gtk+ over the years, leading to rather akward solutions for things one would expect as elementary functionality of any graphic library based on Gtk+, such as theming or geometry-handling.

Gevse

Gevse is a Gnome-based EVSE-system (electric vehicle charging station). It was influenced by similar projects, such as the SmartEVSE. The general aim is the load management and the logging of power consumption for different vehicle charging stations on a single raspberry pi. A lot of the planned functionality is still missing. For the moment it's actual nothing more than a nice little electricity meter with an underlying sqlite-database and some functionality for system maintenance based on Gtk+ and broadway.

Data management: For the moment data logging is just implemented for a simple S0-interfaces. This should be provided in a more general manner in the future.

User management: Users should apply to the charging process via a phone-call, a user interface, automatically via the license plate or by an human administrator. However this functionality is at the moment still completely missing.
