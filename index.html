LibGDV

Description

Working in a scientific environment requires a fast, open and simple approach in software-design. The gtk+ library offers many of these features in terms of UI-programming and the underlying GLib is a powerful approach to use object-oriented programming (OOP) mostly independent of the available platform or programming language. Despite the advantages, most free scientific code does not apply Gtk+ even in applications were the integration would be highly preferable. Gdv tries to provide a part of this functionality.

I startet to work on gdv because I was pretty annoyed by the options that are available for the visualisation of scientific and technical data, such as forcing Gnuplot within a GtkWidget-instance (this approach does work on some platforms and on other it simply doesn't - without any apparent reason), or using really old libraries such as GtkExtra that are now so outdated that they require functions deprecated 17 years ago, or use a rather specialized library such as GOffice that reimplements a lot of the overhead that has been removed from Gtk+ and sometimes applies rather akward solutions for the things one would expect as elementary functionality of any graphic library based on Gtk+, such as geometry-handling.
